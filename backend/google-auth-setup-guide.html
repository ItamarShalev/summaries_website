An interactive guide to creating and storing your web app credentials.

1

Create or Select Your Google Cloud Project

First, you'll need a Google Cloud project to manage your application. This project will house all your APIs, credentials, and settings.

2

Configure the OAuth Consent Screen

The OAuth consent screen is what users will see when they are asked to authorize your application to access their data.

3

Create Web Application Credentials

Now you will generate the specific client ID and client secret that your web application will use to authenticate with Google's APIs.

4

Set Credentials as Environment Variables

For security, you should never hard-code credentials in your code. Store them as environment variables on your computer. This prevents them from being accidentally exposed in your version control history.

On macOS or Linux

To make these variables permanent, add the `export` lines to your shell's startup file (e.g., ~/.zshrc or ~/.bashrc).

export GOOGLE_CREDENTIALS_SUMMARIES_WEBSITE_CLIENT_ID="YOUR_CLIENT_ID"
export GOOGLE_CREDENTIALS_SUMMARIES_WEBSITE_CLIENT_SECRET="YOUR_CLIENT_SECRET"

On Windows

Use the Command Prompt or PowerShell to set your environment variables permanently using the setx command. You must restart your terminal for these changes to take effect.

setx GOOGLE_CREDENTIALS_SUMMARIES_WEBSITE_CLIENT_ID "YOUR_CLIENT_ID"
setx GOOGLE_CREDENTIALS_SUMMARIES_WEBSITE_CLIENT_SECRET "YOUR_CLIENT_SECRET"
